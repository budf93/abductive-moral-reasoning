# Generation of `clutrr0.py`

The file `clutrr0.py` (and similar files like `clutrr1.py`, etc., located in `SAT-LM/tmp/`) is a temporary script generated during the evaluation process of the CLUTRR task.

## Generation Pipeline

### 1. Filename Generation
The names are generated in **`SAT-LM/run_manual.py`** within the `run_evaluation` function. It takes the task name (e.g., "clutrr") and appends the index of the example being evaluated:
```python
filenames = [str(args.task) + str(i) for i in range(len(predictions))]
```

### 2. Orchestration
The filenames are passed into the **`TaskEvaluator.evaluate`** method in **`SAT-LM/task_evaluator.py`**. For CLUTRR, the `CLUTRREvaluator` uses these filenames during the post-processing of the LLM's output.

### 3. Code Execution and File Writing
The actual writing of the Python file to the `tmp/` directory happens in one of two places depending on the prompting style:

*   **SATLM Style**: The `CLUTRREvaluator` calls `clutrr_satlm_exec` in **`SAT-LM/prog_solver/clutrr_solver.py`**, which delegats to `execute_z3_test` in **`SAT-LM/prog_solver/z3_utils.py`**. The `execute_z3_test` function writes the code to `tmp/<filename>.py` and executes it using Z3.
*   **PROGLM Style**: The `CLUTRREvaluator` calls `clutrr_proglm_exec` in **`SAT-LM/prog_solver/clutrr_solver.py`**, which directly writes the file:
    ```python
    if filename:
        os.makedirs("tmp", exist_ok=True)
        with open(os.path.join("tmp", filename + ".py"), "w") as f:
            f.write(output_code)
    ```

## Purpose
These temporary files allow the system to execute the logic generated by the LLM as a standalone script to verify correctness against the Z3 solver or other logical frameworks.
