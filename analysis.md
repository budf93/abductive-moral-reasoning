# Analysis Guide for ARGOS

This guide explains how to use the notebooks in the `./analysis` directory to generate experimental results as described in `readme.txt`.

## 1. Prerequisites: Complete the ARGOS Pipeline
The analysis notebooks process the results generated by the main ARGOS scripts. You must first complete the full data and inference pipeline:

1.  **Logical Extraction**: Run `SAT-LM` on your dataset. Logic programs will be written to `./SAT-LM/tmp`.
2.  **SAT Conversion**: Run `[dataset]_to_sat.py` (e.g., `cluttr_to_sat.py`) to convert logic into SAT format (DIMACS).
3.  **SAT Solving & Backbone Extraction**: Run `cadical_solve.py` then `parse_gen.py` in that order. This generates the SAT solutions and logical mappings (backbones).
4.  **Run ARGOS Inference**: Execute the main method script for your dataset (e.g., `main/argos/cot_met_clutrr.py`). This script performs the inference and saves the final result dictionary.

## 2. Locate Generated Results
The ARGOS scripts save their output (typically a dictionary named `all_outs`) as a `.pkl` file.

*   **Check the Save Path**: In scripts like `main/argos/cot_met_clutrr.py`, look for a line similar to:
    ```python
    pkl.dump(all_outs, open('/path/to/your/results.pkl', 'wb'))
    ```
*   **Update Paths**: You must update the hardcoded `/home/XXXX/` paths in these scripts to a valid directory on your local system before running them.

## 3. Configure and Run the Analysis Notebooks
Once you have generated the `.pkl` result files, open the corresponding notebook in `main/analysis/` (e.g., `analysis_clutrr.ipynb`).

1.  **Update Result Loading Paths**: Locate the cells that load results using `pkl.load()` or `np.load()`. Change these paths to point to your generated `.pkl` files.
    *   *Example*: `outs_70b = pkl.load(open('your_generated_file.pkl', 'rb'))`
2.  **Update Dataset Paths**: Some notebooks load the original dataset for comparison (e.g., using `load_from_disk`). Update `USER_PATH` or specific file paths to point to your local `./SAT-LM/data` directory.
3.  **Execute Cells**: Run the notebook cells sequentially to:
    *   Calculate accuracy and other performance metrics.
    *   Generate histograms (e.g., distribution of steps/exit points).
    *   Produce the final evaluation plots.

## Dataset-Specific Notebooks:
*   `analysis_clutrr.ipynb`: Analysis for the CLUTRR (relational reasoning) dataset.
*   `analysis_folio.ipynb`: Analysis for the FOLIO (first-order logic) dataset.
*   `analysis_pronto.ipynb`: Analysis for the ProntoQA (deductive reasoning) dataset.
